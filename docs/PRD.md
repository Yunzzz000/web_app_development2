# Product Requirements Document (PRD) – 活動報名系統

## 系統概述
這是一個 **活動報名平台**，允許活動主辦方在網站上刊登活動資訊，使用者（潛在參與者）可以瀏覽、搜尋、檢視活動細節並直接報名。系統提供活動分類、搜尋、報名、報名人數統計以及報名成功通知等功能。

## 目標用戶
- 想要參加線上或實體活動的個人使用者。
- 活動主辦單位或社群負責人，需要一個簡易的方式上架活動並管理報名。

## 主要功能
1. **刊登活動說明及報名連結**
   - 主辦方可建立活動頁面，填寫標題、時間、地點、描述、報名截止日、名額上限等資訊。
   - 系統自動產生報名按鈕的連結，使用者點擊後即可填寫報名表單。
2. **分類與歸納活動類型**
   - 依照「講座、工作坊、比賽、社群聚會」等類型分門別類，提供導覽側欄與篩選條件。
3. **搜尋功能**
   - 關鍵字全文搜尋、依日期、地點、類型篩選，支援即時自動完成提示。
4. **活動詳細頁面**
   - 顯示完整活動資訊、圖片、主辦單位聯絡方式、地圖定位、報名人數上限與已報名人數。
5. **報名流程**
   - 使用者填寫簡易表單（姓名、 Email、電話），系統驗證可用性後寫入資料庫，並發送報名成功確認信。
6. **報名人數統計與名額限制**
   - 後端自動跟蹤已報名人數，滿額時自動關閉報名按鈕並顯示「已額滿」訊息。
7. **使用者帳號（可選）**
   - 提供簡易註冊/登入功能，讓使用者可查看歷史報名紀錄與取消報名。

## 非功能需求
- **響應式設計**：支援桌面、平板與手機瀏覽。
- **資料安全**：使用 SQLite（或 PostgreSQL）儲存報名資料，避免 SQL 注入，表單使用 CSRF token。
- **效能**：首頁搜尋與列表在 200ms 內返回（在普通雲端 VM 上測試）。
- **可用性**：系統上線後 99.9% 的可用率，具備基本備份機制。
- **可擴充性**：未來可加入付款流程、活動評論等擴充功能。

## 成功衡量指標
- 首月註冊活動數 >= 10 個。
- 首月活躍使用者（瀏覽或報名） >= 200 人。
- 系統平均頁面載入時間 < 1 秒。
- 報名成功率 >= 95%（除名額滿外的情況）。

## 前提假設
- 使用者具備基本的網路瀏覽能力與 Email。
- 活動主辦方會自行提供活動圖片與描述內容。
- 系統僅在單一語系（繁體中文）環境下運行。

## 限制條件
- 初期僅支援單一租戶（單個網站），未實現多租戶或 SaaS 模式。
- 報名表單欄位固定為名稱、Email、電話，未支援自訂欄位。

## User Stories
1. **活動主辦者**：我要在平台上發布活動，設定活動資訊、圖片、報名上限，系統自動產生報名頁面。
2. **參與者**：我想瀏覽活動列表，依類型或關鍵字搜尋，並點擊報名，收到報名成功的確認信。
3. **系統管理員**：我需要監控報名人數，於名額滿時自動關閉報名按鈕，並產生統計報表。

## Acceptance Criteria
- 活動頁面必須顯示完整資訊、圖片、報名截止日期與剩餘名額。
- 報名表單驗證姓名、Email、電話格式，提交成功後寫入資料庫並寄送確認信。
- 搜尋結果在 200ms 內返回，支援關鍵字、日期、類型篩選。
- 當報名人數達到上限時，報名按鈕變為 "已額滿" 並且無法提交。
- 系統在高峰期 (同時 100 人報名) 仍能保持回應時間 < 1 秒。

## UI / UX Guidelines
- 採用響應式設計，手機、平板與桌面皆有良好排版。
- 活動列表卡片式呈現，包含活動縮圖、標題、日期、類型標籤。
- 報名表單使用簡潔的單行輸入欄位，錯誤提示即時顯示。
- 成功報名後顯示 toast 訊息並寄送 Email。
